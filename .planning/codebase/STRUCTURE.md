**public/**

## Naming Conventions

**Files:**
- kebab-case for config files: `next.config.ts`, `postcss.config.mjs`
- lowercase for app router files: `page.tsx`, `layout.tsx`
- UPPERCASE for documentation: `README.md`

**Directories:**
- lowercase for all directories: `app/`, `public/`, `docs/`
- dot-prefix for hidden: `.next/`, `.claude/`, `.planning/`

**Special Patterns:**
- `page.tsx` - Route page component (App Router convention)
- `layout.tsx` - Route layout wrapper (App Router convention)
- `*.config.*` - Configuration files

## Where to Add New Code

**New Page/Route:**
- Implementation: `app/{route}/page.tsx`
- Layout: `app/{route}/layout.tsx` (optional)
- Tests: `app/{route}/__tests__/` (when testing added)

**New Component:**
- Implementation: `components/{name}.tsx` (create directory)
- Types: Co-located or `types/{name}.ts`
- Tests: `components/__tests__/{name}.test.tsx`

**New API Route:**
- Implementation: `app/api/{endpoint}/route.ts`
- Tests: `app/api/{endpoint}/__tests__/route.test.ts`

**Utilities:**
- Shared helpers: `lib/utils.ts` (create directory)
- Type definitions: `types/` (create directory)

**Hooks:**
- Custom hooks: `hooks/use-{name}.ts` (create directory)
